# Aloysius (5iu5) - Project Portfolio Page

TravelTrio is a desktop travel management application optimized for users who prefer a Command Line Interface (CLI).
It helps organized and thrifty travelers manage complex trip itineraries, track activity-based budgets, and handle currency exchange rates within a single, lightweight interface.

Given below are my contributions to the project.

### **New Feature: Added the ability to add activity to the itinerary**
- What it does: Allows the user to create and store a new activity in the itinerary by specifying details such as the activity name, location, date, and time (start time and end time).
- Justification: This feature is significant to the product because planning a trip requires users to organise their schedules clearly. By allowing activities to be added into the itinerary, the app helps users manage their travel plans in a structured and convenient manner.
- Highlights: This enhancement forms one of the core features of the product, as other features such as listing, deleting, and budgeting for activities depend on activities being created first. In addition, users must first open a trip before they can add activities to it, making this feature closely tied to the overall trip management workflow.

### **New Feature: Added the ability to list all activities in the itinerary**
- What it does: Allows the user to view all saved activities in a clear and organised format, including details such as the activity name, location, date, and time. 
- Justification: This feature improves the product significantly because users need a convenient way to review their planned itinerary. By displaying all activities at once, users can quickly understand their travel schedule, check activity details, and identify any changes that need to be made.
- Highlights: This enhancement is important because it works closely with the add, edit and delete activity features, allowing users to verify that their itinerary has been updated correctly. Its implementation required careful formatting of activity details so that the output is easy to read and consistent. Additional handling was also needed for cases such as an empty itinerary, where the system should respond appropriately instead of displaying invalid or unclear output.

### **New Feature: Added the ability set actual expense for an activity**
- What it does: Allows the user to record the actual amount spent for a specific activity.
- Justification: This feature improves the product significantly because users need a way to track how much they have actually spent during their trip. By recording the actual expense of each activity, users can better monitor their spending and compare it against their planned budget.
- Highlights: This enhancement complements the budgeting feature by allowing users to update activity costs with real spending values. Its implementation required validation to ensure that expense amounts are valid and properly linked to the correct activity. It also strengthens the overall budget management workflow by helping users keep their trip expenses accurate and up to date.

### **New Feature: Added the ability to list expenses for activities**
- What it does: Allows the user to view the expenses recorded for activities in a trip, including the budgeted amount, actual expense, and remaining budget for each activity.
- Justification: This feature improves the product significantly because users need a convenient way to review the financial details of their activities. By listing all recorded expenses in one place, users can better track their spending and monitor whether they are staying within budget.
- Highlights: This enhancement strengthens the budgeting component of TravelTrio by giving users a clearer overview of their activity-related expenses. Its implementation required integrating expense information with the corresponding activities and formatting the output in a clear and readable manner.

### **Enhancements to Existing Features**
- Enhanced the trip listing feature to display the total actual expense incurred for each trip, allowing users to monitor their spending directly when viewing their trips and compare expenses across different trips.
- Added more test cases for existing features to improve reliability and ensure correct behaviour across different usage scenarios.
- Improve the user interface for displaying activities and expenses by presenting them in a structured table format.
